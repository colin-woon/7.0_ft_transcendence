# Product Requirements

## 1. Authentication Module

* **Sign In: (I)**
    * Primary: Google OAuth.
    * Secondary: 42 Intra OAuth (links to student profile).
* **Roles & Permissions: (E)**
    * **Student:** Can view "In Progress" or "Done" projects. Can edit own comments.
    * **Admin (Staff):** Can view ALL projects. Can CRUD (Create, Read, Update, Delete) any forum comment.
* **Profile Management: (D)**
    * View own profile and other user profiles.
    * Status indicators (Online/Offline) visible on profile. 
    * Upload avatar (image file).
    * Edit profile (name, bio, etc.).

## 2. Forum Module (Core Feature)

* **Structure:**
    * Data is flat (comments are equal), potential for nested replies in future iterations.
    * **Markdown Support:** Code blocks must be rendered properly; long blocks minimized by default.
* **Interactions:**
    * ?Searching questions and answers **(?N)**
    * Upvote / Downvote logic.
    * Mention system (`@user`) triggers notifications.
* **Performance:**
    * **Pagination:** Lazy loading (infinite scroll) required for threads.
    * **Updates:** Polling (refresh every ~5 mins) is acceptable for forum updates; Real-time is NOT required for the Forum (reduces complexity).

## 3. Chat Module

* **Relationship Logic: (C)**
    * Users must be "Friends" to chat.
    * Friend Request system (Send, Accept, Deny, Remove).
* **Messaging:**
    * 1-to-1 DM capabilities.
    * "New Message" modal: Search by Intra ID or Name.
    * **UI:** Chat bubbles (Friends on right, User on left).
* **Real-Time Events: (B + J + ?M)**
    * **Typing Indicators:** Must broadcast start/stop typing events.
    * **Read Receipts:** visual indicator when message is seen.
    * **Status:** Live Online/Offline tracking.
